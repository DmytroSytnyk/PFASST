87 ## v0.4.0 -- The Big Improvement (2015/04/17)

94 ### Notable Features
95 
96 * Iteration controll for PFASST via relative and absolute residual tolerances.
97  ([#139][], [#141][], [#142][])
98 
99 * Adding an implicit sweeper with support for LU decomposition of quadrature matrix.
100  ([#150][])
101 
102 * General improvements to the logging and configuration functionality.
103  ([#144][], [#145][], [#146][], [#148][], [#163][], [#168][], [#169][], [#177][])
104 
105 * Successful tests on supercomputers (i.e., JUQUEEN)
106  ([#168][], [#176][])
107 
108 * Improved code organization.
109  ([#156][], [#157][], [#158][], [#170][], [#181][])
110 
111 * Rewamped documentation.
112  ([#149][], [#160][], [#179][], [#183][])
113 
114 * Examples: Boris supports multiple particles and multi-level coarsening
115  ([#152][], [#172][], [#180][])
116 
117 ### Details
118 
119 * logging framework [Easylogging][] has been updated from v9.75 to v9.80.
120  ([#163][])
121 
122 * root finding algorithm for polynomials uses fixed number of iterations or residual tolerance.
123  ([#162][])
124 
125 * reworked how `pfasst::init(...)` is used.
126  ([#148][])
127 
128 * minimum _CMake_ version bumped to 2.8.*6*

131 * `git describe`-based version number as `pfasst::VERSION`

134 * script to generate and compile test coverage report was rewritten in Python 3.x

182 ## v0.3.0 -- The Big Cleanup (2014/12/12)
183 
184 DOI: [10.5281/zenodo.13221](http://dx.doi.org/10.5281/zenodo.13221)
185 
186  $ git diff --stat v0.2.0..v0.3.0
187  > 84 files changed, 7687 insertions(+), 1968 deletions(-)
188 
189 ### Notable Features
190 
191 * Complete rewrite of quadrature functions.
192  ([#103][])
193 
194 * Addition of iteration control mechanism for SDC and MLSDC. _(extention for PFASST is scheduled)_
195  ([#108][])
196 
197 * Introduced framework for passing command line parameters to programs using PFASST++.
198  ([#88][])
199 
200 * Versatile and colourful logging framework based on [Easylogging++][].
201  ([#105][], [#123][])
202 
203 * New Example: Boris-SDC.
204  ([#113][])
205 
206 ### Details
207 
208 * Using [Eigen3](http://eigen.tuxfamily.org/) matrix data types and functionality instead of
209  [Boost's uBLAS](http://www.boost.org/doc/libs/1_57_0/libs/numeric/ublas/doc/index.html).
210  ([#82][], [#83][], [#85][])
211 
212 * Introduced callback interception points in sweepers: `post_step`, `post_sweep` and `post_predict`.
213  ([#107][])
214 
215 * In sweepers: `state(0)` is not assumed to be same as `start_state`.
216  ([#132][])
217 
218 * Examples are now in their own namespace `pfasst::examples::`
219  ([#138][])
220 
221 * Bunch of CMake-related fixes and extentions.
222  ([#83][], [#96][], [#101][], [#109][], [#112][], [#119][], [#121][], [#131][], [#133][], [#135][])
223 
224 * Example of using PFASST++ with a Makefile project (see advection-diffusion example).
225  ([#129][])
226 
227 * Support for [HashDist](https://github.com/hashdist/hashdist) to provide dependencies.
228  ([#129][])
229 
230 * A few corrections and extentions to the documentation.
231  ([#117][], [#120][])
